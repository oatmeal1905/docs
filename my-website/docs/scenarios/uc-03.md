# UC-03 - Отметка об оплате

Организатор фиксирует факт оплаты взноса участником. Участник видит актуальный статус оплаты в личном кабинете.

```plantuml
@startuml
actor "Организатор" as org
participant "Frontend" as fe
participant "Backend API" as api
database "PostgreSQL" as db
participant "Email-сервис" as email

org -> fe : открывает список участников похода
fe -> api : GET /api/trips/{tripId}/applications?status=confirmed
api -> db : SELECT applications WHERE status=confirmed
db --> api : список участников
api --> fe : 200 OK
fe --> org : отображает участников

org -> fe : нажимает "Отметить как оплачено"
fe -> api : PATCH /api/applications/{id}/payment\n{payment_status: paid}
api -> db : проверка: заявка подтверждена?
db --> api : да

api -> db : UPDATE payment\npayment_status=paid
db --> api : OK

api -> email : уведомить участника об оплате
email --> api : OK

api --> fe : 200 OK
fe --> org : статус оплаты обновлён
@enduml
```

## Алгоритм

1. Организатор открывает страницу похода
2. Переходит к списку подтверждённых участников
3. Выбирает участника и нажимает "Отметить как оплачено"
4. Система проверяет - заявка имеет статус `confirmed`?
   - Если нет - возвращает ошибку
5. Обновляет статус оплаты на `paid`
6. Отправляет email участнику

## Предусловия

- Организатор авторизован
- Заявка участника имеет статус `confirmed`

## Постусловия

- Статус оплаты изменён на `paid`
- Участник получил уведомление
- Участник видит статус "Оплачено" в личном кабинете
